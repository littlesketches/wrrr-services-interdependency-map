Victorian Waste, Recycling and Resource Recovery Services Interdependency Map

User guide

About this guide

The WRRR Services interdependency map (the Map) is designed as an interactive experience that contains much of it's core 'guidance' accessible within the interface. This guide provides as extra context and detail that is designed to be used alongside the Map as more comprehensive training materials.

I. The user interface

The Map interface is made up of three panes:

  • Control and info
  • Visualisation
  • Data

Control and info: is where user selections and contextual information is displayed. This pane contains fixed sections for a title and navigation bar (top) and global option switches (bottom). The middle section is dynamic and will display information and selection options, depending on the mode and any user selections.

Visualisation: is where the network visualisation is rendered. The visualisation is interactive, where users can browse and select nodes (entities) using their mouse. The mousewheel can be used to zoom in/out and panning can be accessed with traditional 'click and grab' within the visualisation.

Data: provides summary data statistics that dynamically update with the selected / displayed network. Critically, the Data pane describes and labels the three groupings of of entity (node) types: upstream, intermediate and downstream.

II. Global options

  • Reporting year: a selector for rendering the Map for different available RERCC Plan reporting years is located at the top of the Control and info pane.

  • Interface options: a series of toggle switches are located at the bottom of the Control and info pane. to customise the interface:

    • Tips: is used to show extra information and guidance boxes in the <Control and info for each mode/option. Experienced users may choose to turn these off to reduce the visual clutter, or for print/extracting views.

    • Details: is used to show the detailed marks of the visualisation design (see the Map's key mode > visual details option for more information about these marks). This option allows users to turn of details in situations where they are unnecessary and/or distracting.

    • Labels: is used to show the labels on all intermediate nodes in the visualisaiton. As labelling is necessary for identification but introduces considerable visual clutter, this option is available to suit individual user needs.

III. Navigating modes and options

The navigation menu bar is located under the title in the Control and info pane. There are five modes (that can be thought of as sections) that can be selected in any order. Most modes contain options that are designed for different purposes.

1. Overview

This is serves as a landing page and introduction to the Map. The overview is brief but ends by pointing users to the breakdown/grouping of nodes shown in the Data pane. While subtle (and constrained by space), this should be called out in a training situation (and for anyone reading this guide as training). Moreover, it is an important to understanding that entities — which include all Responsible Entities and their identified suppliers and cusomters — are grouped into:

  • upstream: where they only have connections 'out' to customers. These are referred to in network terms as 'source' nodes

  • downstream: where they only have connections 'in' from suppliers. These are referred to in network terms as 'sink' nodes

  • intermediate: where they only have connections 'in' from suppliers. These are referred to in network terms as 'sink' nodes

The more precise (but potentially vague) network terms are used in the interface as they are more concise and preferred to optimise information density.

2. Entities

This mode focuses on exploring the supplier-customer connections from a selected or selection of entities. The entities mode contains the following options:

  • Responsible: This option is designed to help users locate and select a specific Responsible Entity to interrogate.

  • Browse: This option allows users to view the network connections for any entity, by hovering their mouse over them in the visualisation. It is designed as a visual exploration tool where the connected networks of entities can be very quickly viewed and compared, without the need to select and deselect with mouse clicks.

  • Trace: This option allows users to more deeply examine the network connections emanating from any entity. Users are instructed to select a single entity from the visualisation (with a mouse click), which will initially only show the entities immediate upstream and downstream connections. The user can then drill deeper into the network by repeatedly clicking on that node. This interface is designed to provide users with a way to control and view how an entities connections are propagated through the network (i.e. the spread of its 'influence').

  • Blend: This option allows users to combine the connected networks of multiple entities. Any mutual upstream or downstream connections are identified and shown. This option is useful for looking at the network coverage or the shared and exclusive connections of multiple entities.

3. Service

This mode focuses on using meta information about "services' available to filter the view of the network. The services mode contains the following options:

  • Essential waste services: Allows users to filter the network by connections to Responsible Entities that have been assigned (by DEECA/RV) to the selected essential waste services.

  • Customer and supplier services: Allows users to filter the network by connections to entities that have been assigned (by Responsible Entities and reviewed by DEECA/RV) to the selected services. Services selections are broken into sections for upstream, intermediate and downstream entities (with only relevant/available services shown).

  • Service frequency: Allows uses to filter the network by connections of the same identified frequency.

4. Vulnerability

This mode focuses on using meta information about the vulnerability of supplier-customer relationships to filter the view of the network. The vulnerability mode contains the following options:

  • Failure points and specialised: Allows the user to filter the network by connections that have been identified by Responsible Entities are belonging to either of these criteria. These views can each be turned on from their respective switches, but only one is shown at a time.

  • Feedback: Allows the user to filter the network by connections that correspond to three feedback types. These views can each be turned on from their respective switches, but only one is shown at a time.

5. Key

This mode provides guidance on how the visual marks in the Map's visualisation represent the RERCC Plan data. The Key mode contains the following options ('sections') that are self-documented:

  • Node and links: explains how entities and their connections are rendered.

  • Upstream and downstream: explains how colours are used to encode information about the selected entity, and which connections (links and nodes) are upstream, downstream or both from the selection.

  • Visual details: explains how entity and link meta-data is encoded as detailed visual marks in the Map's visualisation.

IV. Keyboard shortcuts

A set of keyboard shortcuts are provided to assist with quick navigation and add custom functionality to the Map. These are provide for advanced users.

  • t
    Toggles the tips switch
  • d
    Toggles the details switch
  • l
    Toggles the labels switch
  • esc
    Resets any entity selection (i.e. deselects).

Pane view

  • [
    Used to hide/unhide the Control and info pane
  • ]
    Used to hide/unhide the Data pane
  • \
    Used to hide/unhide the content of the Control and info pane, leaving the title (useful as an option for print/extract)
  • p
    Used to hide/unhide both the Control and info and Data panes at the same time (useful as an option for print/extract)

V. Use in other mediums

PDF / image extraction and printing

The Map visualisation is rendered as an SVG (Scalable Vector Graphic) that is resolution-less (aka a vector graphic), meaning that it can be scaled to any size (on screen) without losing clarity. Vector graphics are best preserved in extraction transfer to other documents as a PDF (i.e. which supports vector graphics).

Accordingly, the recommended way to extract the Map is through your web browsers print/save to PDF feature.

A set of view modes, accessible via keyboard shortcuts (see above) can also help to setup a suitable view for extraction. These can be combined with global options for showing visualisation details and labels for more customised views to extract.
