Victorian Waste, Recycling and Resource Recovery Services Interdependency Map

Data preparation checklist

About this checklist

The WRRR Services interdependency map (the Map) is designed to load in new data for each RERCC Plan reporting year. The latest years data is the 'default' view for the Map, however historical data is available and can be rendered using the Map's reporting year selector. .

DEECA is responsible for adding new reporting year data to the tool, with detailed guidance provided in separate documents (delivered in June 2025).

This checklist provides a summary of the steps DEECA need to undertake to prepared and load data each year. They are written for data administrators and use terminology that is explained in the aforementioned (separate) documentation, which should be consulted for further details.

  1. Use the Data Input Model: as a template for preparing the ten data input tables. The Data Input Model (documented) is managed by DEECA. The original template with documentation is available as a Google Sheet here (with anonymised data)

  2. Date cleansing: Ensure all link data (the source data) is cleaned and structured in accordance with the data input tables. All options/entries reported in structured fields must appear in the appropriate schema tables.

  3. Create node table: Ensure entities included in link data are in the node-master table. Each Responsible Entity must be flagged and assignment to essential waster services made.

  4. Copy Excel Data Input Model into the package for data extraction : A copy of the Excel spreadsheet with all prepared data input tables should be copied into the _data-preparation/spreadsheet folder of ths application package.

  5. Use the Python script to convert/package data to a javascript : The script must be edited to with the appropriate reporting year, and the path and filename to the source Excel spreadsheet checked. Once executed, the script produces the data input file (data-xxxx.js).

  6. Move the data-xxxx.js file into the application folder : Data files for the application are located in the _static/js/data/by-reporting-year/ folder. New data files can be added to overwrite the placeholder files for each year. These files (to 2030) are already linked to the visualisation.

  7. Open the visualisation to check that new data is available: Successfully loaded data will appear as the default (if as expected, it is for the 'latest year').
